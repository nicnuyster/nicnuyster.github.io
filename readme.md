# Персональное портфолио / CV-сайт

Одностраничный адаптивный веб-сайт, предназначенный для представления профессионального опыта, образования, проектов и контактной информации в интерактивном и современном формате. Проект полностью построен на нативных веб-технологиях без использования сторонних фреймворков, что обеспечивает высокую производительность, простоту сопровождения и легкость развертывания.

### Frontend

# 2. Общее описание проекта

Проект представляет собой современное одностраничное портфолио, выполняющее роль интерактивного онлайн-резюме.

Вместо классического длинного документа информация разделена на полноэкранные секции, между которыми пользователь перемещается при помощи прокрутки страницы или панели навигации, расположенной справа.

Сайт состоит из пяти основных разделов:

- О себе
- Академический опыт
- Коммерческий опыт
- GitHub-проекты
- Контактная информация

Особое внимание уделено пользовательскому опыту: реализованы плавные анимации, динамическое изменение фонового градиента, карусель изображений, адаптивная верстка и интерактивная навигация между разделами.

Поскольку проект полностью основан на HTML, CSS и JavaScript, он не требует установки зависимостей и может быть размещен практически на любом статическом хостинге.

# 3. Подробное описание проекта

## Архитектура проекта

Проект имеет простую и понятную структуру.

```
project/
│
├── index.html
├── style.css
├── script.js
└── images/
```

Каждый файл отвечает за отдельную область функциональности.

### HTML

HTML-документ определяет структуру всего приложения.

Основные компоненты страницы:

- динамический градиентный фон;
- вертикальная навигационная панель;
- контейнер со скроллом;
- пять полноэкранных секций;
- карусель изображений;
- карточки академического опыта;
- карточки коммерческого опыта;
- карточки GitHub-проектов;
- блок контактной информации.

Каждая секция использует семантическую HTML-разметку и имеет собственные идентификаторы и атрибуты, которые используются JavaScript для навигации и управления анимациями.

---

## CSS

Файл стилей отвечает за внешний вид и адаптивность интерфейса.

### Основные возможности

- глобальный CSS Reset;
- использование CSS-переменных;
- адаптивная верстка;
- стеклянный стиль интерфейса (Glassmorphism);
- адаптивная типографика;
- поддержка мобильных устройств;
- эффекты наведения;
- плавные анимации и переходы.

### Используемые технологии верстки

Для построения интерфейса используются:

- Flexbox;
- CSS Grid;
- Scroll Snap для навигации между полноэкранными секциями.

Интерфейс автоматически адаптируется под:

- настольные компьютеры;
- планшеты;
- мобильные устройства.

---

## JavaScript

Файл JavaScript реализует всю интерактивную логику приложения.

Функциональность разделена на несколько независимых компонентов.

### 1. Навигация между разделами

Система навигации:

- определяет активную секцию страницы;
- подсвечивает соответствующий индикатор;
- обеспечивает плавную прокрутку между разделами;
- синхронизирует навигацию со скроллом пользователя.

Для определения текущего раздела используется **Intersection Observer API**.

---

### 2. Динамический фон

Каждой секции назначен собственный основной цвет.

Во время прокрутки страницы JavaScript плавно интерполирует цвета между соседними разделами, формируя анимированный градиент без резких переходов.

Для вычисления промежуточных цветов используется интерполяция RGB-компонентов.

---

### 3. Карусель изображений

Раздел **«О себе»** содержит интерактивную карусель изображений.

Поддерживаются следующие возможности:

- переключение вперед и назад;
- автоматическая смена изображений;
- индикаторы текущего слайда;
- управление с клавиатуры;
- поддержка сенсорных устройств;
- автоматическая пауза при наведении;
- автоматическое возобновление воспроизведения.

Карусель работает по циклическому принципу.

---

### 4. Управление отображением секций

JavaScript отслеживает степень видимости каждой секции с помощью **Intersection Observer**.

Это позволяет:

- определять активный раздел;
- синхронизировать состояние навигации;
- управлять отображением контента.

Подобный подход снижает нагрузку на браузер по сравнению с постоянной обработкой событий прокрутки.

---

### 5. Оптимизация производительности

Для обеспечения плавной работы используются следующие подходы:

- requestAnimationFrame для обработки прокрутки;
- passive event listeners;
- Intersection Observer вместо постоянного отслеживания scroll;
- CSS-анимации вместо JavaScript-анимаций;
- кэширование ссылок на DOM-элементы;
- минимизация количества изменений DOM.
